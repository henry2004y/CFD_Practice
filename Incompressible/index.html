Incompressible Fluid Solver

Fluid simulation in industry focuses mostly on incompressible solvers. Mathematically incompressible fluids have simpler expressions and can be solved numerically in an efficient way. Here I am trying to learn some of the basic ideas and tricks, following the book Fluid Simulation for Computer Graphics, 2nd Edition and the excellent implementation. This is also a great opportunity for me to practice C++.

Eulerian approach is much easier in numerically approximating the spatial derivatives.

Semi-Lagrangian method is generally used for advection in solving the incompressible fluids numerically, because it's fast and often unconditionally stable.

Splitting Method

With operator splitting, we'll work out methods for solving these simpler equations:

DqDt=0(advection),∂u∂t=g(body forces),∂u∂t+1ρ∇p=0(pressure)such that ∇⋅u=0(incompressibility)\begin{aligned} \frac{Dq}{Dt} &= 0 \qquad (\text{advection}), \\ \frac{\partial \mathbf{u}}{\partial t} &= \mathbf{g} \qquad (\text{body forces}), \\ \frac{\partial \mathbf{u}}{\partial t} + \frac{1}{\rho}\nabla p &= 0 \qquad (\text{pressure}) \\ \text{such that}\, \nabla\cdot\mathbf{u} &= 0 \qquad (\text{incompressibility}) \end{aligned}DtDq​∂t∂u​∂t∂u​+ρ1​∇psuch that∇⋅u​=0(advection),=g(body forces),=0(pressure)=0(incompressibility)​

Basic fluid algorithm:

  • Start with an initial divergence-free velocity field u0\mathbf{u}^0u0

  • For time step n=0,1,2,...n=0,1,2,...n=0,1,2,...

    • Determine a good time step Δt\Delta tΔt to go from time tnt_ntn​ to tn+1t_{n+1}tn+1​.

    • Set uA=advect(un,Δt,un)\mathbf{u}^A = \text{advect}(\mathbf{u}^n,\Delta t, \mathbf{u}^n)uA=advect(un,Δt,un).

    • Add uB=uA+Δg\mathbf{u}^B = \mathbf{u}^A + \Delta \mathbf{g}uB=uA+Δg.

    • Set un+1=project(uB,Δt)\mathbf{u}^{n+1} = \text{project}(\mathbf{u}^B,\Delta t)un+1=project(uB,Δt).

MAC Grid

In the early days of computational fluid dynamics, Harlow and Welch introduced the marker-and-cell (MAC) method method for solving incompressible flow problems. One of the fundamental innovations of this paper was a new grid structure that makes for avery effective algorithm for enforcing incompressibility, though it may seem inconvenient for everything else.

The so-called MAC grid is a staggered grid, i.e., a grid where the different variables are stored at different locations. (I remember there is another name of staggered grid after people doing MHD for EM field, but it's basically the same thing.)

Level Set Method

This part needs more complete info!

Some geometric problems are extremely common in fluid solvers:

  • When is a point inside a solid?

  • What is the closest point on the surface of some geometry?

  • How do we extrapolate values from one region into another?

When I wrote my simple field tracer in Julia, I spent some time on the first query especially for 1D and 2D.

The first query above suggests the right approach is implicit surfaces. While there are potentially a lot of ways to generalize this, we will focus on the case where we have a continuous scalar function φ(x,y,z) which defines the geometry as follows:

  • a point x is outside the geometry when φ(x) > 0,

  • it’s inside the geometry when φ(x) < 0, and

  • a point x is on the surface when φ(x) = 0.

In some other contexts within computer graphics, the convention may be reversed (the “outside” is the negative region) or a threshold value otherthan zero could be used, but this is usually the way we do it in fluid simulation.

Signed distance function (SDF): the gradient of the function is exactly unit-length normal on the surface.

level set: a signed distance function that has been sampled on a grid.

Advection

In the step of advection,

DqDt=0, \frac{D q}{Dt} = 0, DtDq​=0,

we will encapsulate this in a numerical scheme

uA=advect(un,Δt,qn) \mathbf{u}^A = \text{advect}(\mathbf{u}^n,\Delta t, q^n) uA=advect(un,Δt,qn)

It bears repeating here: advection should only be called with a divergence-free velocity ﬁeld u\mathbf{u}u, i.e., one meeting the incompressibility constraint, which also satisfies the required boundary conditions.

Projection

The heart of this scheme is to make this fluid incompressible and simultaneously enforce boundary conditions. The implementation of

un+1=project(uB,Δt) \mathbf{u}^{n+1} = \text{project}(\mathbf{u}^B,\Delta t) un+1=project(uB,Δt)

is the central topic.

The project routine will subtract the pressure gradient from the intermediate velocity field u\mathbf{u}u,

un+1=un−Δt1ρ∇p, \mathbf{u}^{n+1} = \mathbf{u}^n - \Delta t \frac{1}{\rho}\nabla p, un+1=un−Δtρ1​∇p,

so that the result satisfies incompressibility inside the fluid

∇⋅u=0, \nabla\cdot\mathbf{u} = 0, ∇⋅u=0,

and satisfies the solid wall boundary conditions

un+1⋅n^=usolid⋅n^at solid boundaries, \mathbf{u}^{n+1}\cdot\widehat{n} = \mathbf{u}_{\text{solid}}\cdot\widehat{n}\qquad \text{at solid boundaries,} un+1⋅n=usolid​⋅nat solid boundaries,

while also respecting the free surface boundary condition that pressure be zero there:

p=0at free surfaces. p = 0\qquad \text{at free surfaces.} p=0at free surfaces.
